chapter 1  线性表

一 、 顺序表
1.在线性表中中，逻辑上相邻的元素在物理存储位置上也同样相邻

2.可以按照元素的位序号进行随机存取

3.进行插入，删除操作时需要移动大量元素

4.需要进行存储空间的预先分配，可能会造成空间浪费，但是存储密度较高

插入和删除操作的效率很低，因为每次进行删除或者插入，元素的移动次数都很多

数组的容量不可更改

容量过小会造成溢出，容量过大会造成资源浪费

二、链表
1.单链表
2.循环链表:简化某些操作
3.双向链表:减小时间复杂度

三、顺序表和链表比较

顺序表 :
    优点 :
        1.可进行高效随机存取
        2.存储密度高,空间开销小
        3.实现简单,便于使用
    缺点 :
        1.需要预先分配存储空间
        2.不便于进行插入和删除操作

链表 :
    优点 :
        1.灵活,可以进行存储空间的动态分配
        2.插入，删除效率高
    缺点 :
        1.存储密度低
        2.不可按照位序号随机存取

小结
    1.线性表是其组成元素之间具有线性关系的一种线性结构,其实现方式主要为顺序存储的实现和基于链式存储的实现
    2.线性表的顺序存储结构成为顺序表,可用数组实现,可对数据元素进行随机存,时间复杂度为O(1),在插入或者删除元素时时间复杂度为O(n)
    3.线性表的链式存储结构成为链表,不能直接访问给定位置上的数据元素,必须从头结点开始沿着后继结点进行访问,时间复杂度为O(n),在插入或删除元素时不需要移动任何元素
    ,只需要更改结点的指针域即可,时间复杂度为O(1)
    4.循环链表将链表的首尾相连,即尾结点的指针域尾指向头结点的指针,从而形成了一个环状的链表
    5.双向链表的结点具有两个指针域,一个指针指向前驱结点,一个指针指向后继结点,使得查找某个结点的前驱结点不需要从表头开始顺着链表进行查找,减小时间复杂度
